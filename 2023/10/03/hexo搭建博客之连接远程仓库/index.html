hexo搭建博客之连接远程仓库

发表于2023-10-03|更新于2025-10-22|hexo
|阅读量:

第一步 git 安装与配置

第二步 创建github page，并配置

1、登陆github（没有自己创建，邮箱可以用outlook,学校机房其他邮箱可能会登不上）
2、创建一个新的仓库，仓库名必须按照以下格式

例如：我的用户名为Impurity778,则我创建的仓库名为Impurity778.github.io(记得勾选add readme.md)
仓库须设置为公开

第三步 配置文件_config.yml（在你用hexo创建的的blog目录下）

1、使用token（较为麻烦，每次都要复制并粘贴）


右上角点击头像，点开“settings”

左面的侧边栏中找到developer settings

选择fine-grained,并创建
记得把下面的权限都钩上你看的懂的话当然可以自选!!!注意最好只有一个token
创建好后记得把token保存下来

2、使用ssh进行访问（推荐的做法，但配置较为繁琐）

  • 1 配置自己的git信息（前面已经做过的可跳过）
  • 2 生成ssh key（可以直接生成在项目目录里，也可以在父目录直接生成）
   ssh-keygen -t rsa -C "your email for github here"
  • 3 找到给出的ssh密钥文件 .ssh/id_rsa和 .ssh/id_rsa.pub（在/.ssh目录中）
  • 4 在GitHub的Setting里面，找到SSH keys，把id_rsa.pub里面的内容全部复制到key中，title随便即可
  • 5 在之前讲到的_config,yml中进行编辑，拉到最底部的deploy:进行编辑：
deploy:
  type: git
  repo: git@github.com:your github name/your github name.github.io.git #像这样git@github.com:Impurity778/Impurity778.github.io
  branch: main  #分支的选择去github上看自己的
  • 6 设置url 找到_config.yml中的url,把自己仓库的地址粘帖在后面即可
  • 7 测试ssh连接
ssh -T git@github.com


若出现如图所示的提示，表示连接成功
